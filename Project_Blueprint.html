Development Phases
01
Abstraction
The Foundation
Translated real-world entities into code. Customer, Show, and Room defined as public classes — the blueprint stage. Single object instantiation in main() as proof of concept. No security, no advanced features; just the raw foundation.
Classes Objects Methods Basic I/O
02
Encapsulation
The Vault
Private attributes with validated public setters acting as security checkpoints. Phone validated to 10-digit constraint. State transitions controlled via bookSeat() — the object now actively protects its own data.
Private Members Getters/Setters Data Hiding Validation
03
Collections
The Database
Global vector<T> containers replace single-object instantiation. push_back() for insertion, range-based loops for traversal, iterator-based erase() for deletion. Do-while menu loop with switch-case routing completes the interactive shell.
std::vector Iterators push_back Menu System
04
Inheritance
The Family Tree
Booking established as protected base class. MovieBooking and HotelBooking derived with constructor delegation and overridden display(). The virtual keyword unlocks runtime polymorphism — each child prints its own receipt format without rewriting the base logic.
Base/Derived public Inheritance Method Overriding Constructor Delegation
05
Integration
The System
All phases converge. bookMovie() and bookHotel() engines validate customer IDs, check seat/room availability, calculate totals, and instantiate the correct booking type — proving encapsulated entities, sequence containers, and inherited records can wire together into a functioning application.
Object Interaction System Architecture Polymorphic Iteration Revenue Reporting
