## About the Project

This system manages both a cinema hall and a hotel under one roof. Customers can book movie tickets, reserve hotel rooms, or do both. The system tracks customers, shows, rooms, and all bookings using proper object-oriented design.

The project was developed progressively across 6 phases — each phase introduces a new OOP concept as taught in class, building on top of the previous version.
This project was built iteratively to demonstrate a strong command of core Object-Oriented Programming (OOP) principles. By utilizing modern C++ standard libraries and efficient memory management, the entire fully-featured system has been optimized to run in under 200 lines of code.


## OOP Principles Demonstrated (Grading Alignment)

This application was engineered specifically to satisfy the assignment rubric, demonstrating the following concepts:

* **[CO1] Classes & Objects:** Entities like `Customer`, `Show`, and `Room` are modeled as distinct classes, serving as blueprints for the interactive objects used throughout the system.
* **[CO2] Encapsulation & Data Hiding:** All sensitive attributes (e.g., IDs, names, prices) are marked as `private` or `protected`. Data modification and retrieval are strictly controlled through inline getter and setter functions.
* **[CO3] Sequence Containers (Arrays of Objects):** Replacing traditional fixed C-style arrays, the system uses `std::vector` to dynamically manage memory. This allows for infinite, scalable collections of customers and bookings without hardcoded limits or complex array-shifting logic.
* **[CO4] Inheritance & Polymorphism:** The system features a base `Booking` class that holds shared data (Booking ID, Customer Name, Total Amount). The `MovieBooking` and `HotelBooking` classes inherit from this base class, promoting extreme code reuse. Both derived classes override a `virtual display()` method to output polymorphic receipts.

## Features & Logical Flow

1. **Customer Management:** Register new customers into the system database. Customers must exist in the system before a booking can be made.
2. **Inventory Management:** Add new Movie Shows (with specific seating capacities) and Hotel Rooms (with specific nightly rates).
3. **Smart Booking Engine:** * The system cross-references the customer ID and the requested inventory.
   * It validates seat availability for movies and booking status for rooms.
   * It dynamically calculates total costs (e.g., `Price * Tickets` or `Price * Nights`).
4. **Revenue Tracking:** A single command outputs all registered entities and sums up the total polymorphic revenue generated by both the cinema and the hotel.

### Prerequisites
* A standard C++ compiler (e.g., GCC, Clang, or MSVC).
* C++11 standard or higher (required for range-based for loops and `std::vector`).

### Compilation Instructions (Terminal)
1. Open your terminal or command prompt.
2. Navigate to the directory containing the source code.
3. Compile the file using g++:
   ```bash
   g++ -std=c++11 phase5_final.cpp -o BookingSystem
   ```
4. Run the executable:
   * **Windows:** `BookingSystem.exe`
   * **Mac/Linux:** `./BookingSystem`
